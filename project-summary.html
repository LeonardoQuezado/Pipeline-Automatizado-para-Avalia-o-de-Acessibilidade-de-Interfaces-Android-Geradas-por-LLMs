1. Visão Geral

O Accessibility LLM Pipeline é uma ferramenta de pesquisa desenvolvida como tema de dissertação de mestrado. Seu objetivo é investigar a capacidade dos grandes modelos de linguagem (LLMs) em gerar código Android acessível, medindo automaticamente o resultado com análise estática de acessibilidade.

A ferramenta recebe um prompt descrevendo uma tela Android, envia esse prompt simultaneamente a múltiplos LLMs, compila o código gerado em um APK real e executa o Android Lint com um conjunto de regras de acessibilidade selecionadas, produzindo um relatório comparativo entre os modelos.

Todo o ambiente é containerizado com Docker e expõe uma interface web acessível em http://localhost:8000, sem necessidade de instalar Android Studio, SDK ou JDK na máquina do pesquisador.

2. Fases do Pipeline

Fase 1
Entrada do Prompt
Pesquisador descreve a tela via interface web ou arquivo prompt.txt. Instruções técnicas fixas são anexadas automaticamente.
Fase 2
Geração de Código
Prompt enviado em paralelo para Gemini, OpenAI e Anthropic via SDK assíncrono (asyncio.gather). Cada LLM retorna Kotlin + XML.
Fase 3
Extração
Regex extrai os dois blocos de código (```kotlin e ```xml) da resposta em markdown de cada LLM.
Fase 4
Build do APK
Código injetado em projeto-template. Gradle executa: kotlinc → AAPT2 → D8 → APK Builder. APK real gerado no container.
Fase 5
Análise Lint
Android Lint analisa o APK + fontes com 12 regras de acessibilidade. Resultados exportados em XML e HTML.
Fase 6
Relatório
Pipeline filtra apenas os problemas de acessibilidade, conta por severidade e exibe resumo comparativo entre os LLMs na interface web.

Os três modelos são chamados em paralelo com asyncio.gather(). LLMs sem chave de API configurada são ignorados silenciosamente. A interface web exibe badges coloridos indicando quais modelos estão disponíveis em tempo real via GET /api/status.

5. Análise de Acessibilidade (Android Lint)

O Android Lint é a ferramenta de análise estática oficial do ecossistema Android. Ele opera em dois domínios complementares:

  • AST Kotlin/Java — analisa a árvore sintática do código-fonte, verificando chamadas de API, configurações de listeners e uso correto de atributos programáticos.
  • DOM XML — percorre os arquivos de layout, verificando atributos como contentDescription, tamanhos de toque, contraste de cores e rótulos.

O pipeline configura 12 verificações de acessibilidade com severidade personalizada em app/lint.xml:

6. Pontos Fortes e Limitações

Pontos Fortes

  • Pipeline totalmente automatizado: do prompt ao relatório, sem intervenção manual
  • Ambiente reproduzível via Docker — funciona em qualquer máquina
  • Comparação direta entre 3 LLMs no mesmo prompt
  • APK real compilado e testado — não apenas análise de texto
  • Interface web com terminal ao vivo e indicadores visuais
  • Separação entre prompt variável e instruções técnicas fixas
  • Biblioteca de prompts pré-definidos para cenários comuns
  • Relatório HTML detalhado gerado pelo Lint para análise profunda

Limitações Atuais

  • Apenas código de uma Activity por execução
  • Lint estático não detecta problemas de acessibilidade em tempo de execução
  • LLMs ocasionalmente geram código com erros de compilação
  • Sem persistência histórica das execuções
  • Análise limitada ao sistema de Views Android (sem Compose)
  • Custo de API por execução (especialmente GPT-4o)
  • Build lento (~1-3 min por LLM) devido ao Gradle

8. Trabalho Futuro

  • 📱
    Google Accessibility Scanner Integrar o Accessibility Scanner (via UIAutomator) para análise dinâmica em emulador Android — complementando a análise estática do Lint com verificações em tempo de execução.
  • 📊
    Histórico de execuções e métricas comparativas Banco de dados SQLite para armazenar resultados de cada execução e gerar gráficos comparativos entre LLMs ao longo do tempo — essencial para a análise estatística da dissertação.
  • 🔄
    Regeneração iterativa Enviar os problemas de acessibilidade encontrados de volta ao LLM e pedir correções automaticamente — medir se os modelos conseguem auto-corrigir código inacessível.
  • 📄
    Exportação de relatório PDF Geração automática de relatório PDF comparativo ao final de cada execução, adequado para anexar à dissertação.
  • 🎨
    Suporte a Jetpack Compose Expandir o pipeline para gerar e analisar código Compose (novo toolkit de UI do Android), além do sistema tradicional de Views.
  • 🔍
    Análise de múltiplas Activities Suporte a prompts que descrevem fluxos completos com múltiplas telas, gerando e analisando vários arquivos de layout por execução.
